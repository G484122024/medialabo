「Webコンテンツ作成」の成果

本テーマについて

このテーマでは、HTMLでページの見出しやリストを作り、CSSでデザインをきれいにする方法を学びました。さらに、JavaScriptを使ってページに動きをつける方法も勉強しました。このページは、本テーマの学習成果を表しています。

プロジェクト課題について

このプロジェクトは、世界の天気を調べるウェブページです。使い方はとても簡単で、都市のIDを入力するか、一覧から都市を選ぶと、その都市の天気がすぐに表示されます。表示される情報は、都市の名前、天気、気温（最高・最低）、湿度、風速、風向、そして緯度と経度です。天気の情報はカードのような見た目で、見やすくなっています。ページの色は青と白のグラデーションです。また、都市の一覧はマウスでクリックすると開き、他の都市も簡単に選べます。このプロジェクトは、JavaScript や CSS を使って作られており、プログラミングの勉強にも役立ちます

プロジェクト課題で作成したページ: project.html

数当てゲーム

JavaScriptを使って、数当てゲームのプログラムを作りました。1から10までの数字の中からランダムに正解の数字が決まります。ユーザーはテキストボックスに数字を入力して、ボタンをクリックします。入力した数字が正解と同じなら、「正解です！」とコンソールにメッセージが出ます。まちがっていると、「ちがいます。もっと大きいです」や「もっと小さいです」というヒントが出るようにしました。ヒントを見ながら、何回もチャレンジできます。JavaScriptの if 文やイベント処理、Math.random()の使い方などを勉強できました。シンプルですが、とてもおもしろいゲームになりました。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

HTML のきほんタグをべんきょうしました。h1 や h2 をつかって、みだしをつくりました。ul や ol、li をつかって、リストをつくることができます。img タグをつかえば、がぞうをひょうじできます。a タグでリンクもはれます。dl、dt、dd タグをつかって、せつめいリストもつくれます。table タグをつかうと、ひょうもひょうじできます。いろいろなタグをつかうと、きれいでわかりやすいページがつくれます。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSを使ってHTMLのページをきれいにする方法を勉強しました。CSSを使うと、文字の色や大きさを変えたり、背景の色をつけたりできます。文字を太くしたり、フォントを変えたりもできます。自分のページを見やすくできます。

演習の成果: 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

JavaScriptの変数、条件分岐、イベント処理の基礎を学びました。JavaScriptを使って、数当てゲームのプログラムを作りました。Firefoxの開発者ツールでデバッグのやり方を勉強しました。コンソールを使って変数の値を見たり、ブレークポイントを使ってプログラムの動きを一行ずつ確認したりしました。

演習の成果: 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

「イベント処理とDOM」の基本を学びました。まず、input要素やbutton要素を使って、ユーザーインターフェースを作成しました。そして、querySelectorで要素を取得し、addEventListenerでイベントを登録する方法を実践しました。JavaScriptでは、ボタンをクリックしたときにテキストボックスの値を取得して、コンソールに表示する方法や、条件分岐で動きを制御する方法も学びました。最後に足し算ゲームや数当てゲームでDOM操作を練習しました。

演習の成果: 演習4-10: 足し算ページ

第5回 イベント処理

JavaScriptを使ったDOM操作について学びました。特に、HTML要素をJavaScriptで新しく作成し、ページ上に表示する方法を練習しました。課題では、天気データをページに表示する printDom() 関数を作成し、都市名、気温、湿度などの情報を<div>要素内に出力しました。また、ボタンをクリックすると結果が表示されるようにイベントリスナーの設定も行いました。

演習の成果: 演習5-7: DOMプログラミング(2)

レポート

工夫点：私はこの作品でユーザーが使いやすいページを作ることに力を入れました。天気を調べるための入力ボックスだけでなく、都市のリストも追加しました。なぜなら、都市のIDがわからないと天気が調べられないからです。リストの都市をクリックすると、その都市の天気がすぐに出るようにしました。また、CSSを使って、マウスをリストにのせたときだけ表示されるようにして、ページがごちゃごちゃしないように工夫しました。こうすることで、見やすくて使いやすいページになったと思います。

苦心点：いちばんむずかしかったところは、JavaScriptで検索ボタンを何回も押したときに、前の結果が残ってしまうことでした。最初は、新しい結果が前の結果の下にどんどん出てきて、ページがとても見にくくなっていました。数あてゲームを作ったときも同じような問題がありました。そこで、検索をするときに前の結果を消すようにして、いつも新しい結果だけが出るようにしました。この処理を作るのはむずかしかったですが、自分でいろいろ調べて何回も試して、うまくできたときはとてもうれしかったです。

強調点：デザインの部分でも工夫しました。たくしょくだいがくのページでは、大学のイメージカラーのオレンジ色を使いました。天気のページでは、空や天気をイメージしたうすい青色を使いました。たくさん文字があっても目がつかれにくいように、シンプルで読みやすいレイアウトにしました。ボタンや文字は、まるい形にして、やさしい印象になるようにしました。また、リンクをクリックしたあとでも色が変わらないようにして、どこを見たかわかりやすくしました。

感想：このプロジェクトをとおして、HTML、CSS、JavaScriptの使い方をもっと深く理解できました。前にも少しだけWebページを作ったことがありましたが、今回は自分でアイデアを考えて、実際に使えるページを作ることができたので、自信になりました。作っているときにエラーが出て、なかなかうまくいかないこともたくさんありましたが、自分で調べたり先生に聞いたりして、１つずつ直しました。ページがちゃんと動いたときは、大きな達成感がありました。

これからは、今回学んだことを使って、もっとむずかしいWebページや、うごきのあるページにもチャレンジしたいです。時間があれば、天気の情報をグラフで表示したり、スマホでも見やすいデザインも作ってみたいです。
